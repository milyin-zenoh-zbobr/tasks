For zenoh-c project: 
- Never edit files in the include directory, they are autogenerated from rust
- Pay attention to coherent usage of `z_`, `zc_`, `ze_` prefixes depending on function.
- use `z_sleep_s` instead of `sleep` which doesn't work on windows

For zenoh-pico project: Never edit files include/zenoh-pico.h, include/zenoh-pico/config.h, library.json and zenohpico.pc

For zenoh-cpp project: Update the zenoh-c submodule to latest version to keep up to date to zenoh-c API changes

For all rust projects: 
- avoid using fully qualified types/functions, like `foo_crate::bar_mod::BuzzType`. Add `use foo_crate::bar_mod` to the header of the file. Even for feature-gated code prefer use feature-gated `use` in the header. But make exception to this rule if there are other usage of this type/function in the same file in the fully qualidied form already exists - follow the existing pattern in this case

For all projects: the zenoh serialization / deserialization api support containers, use them, do not store the size by yourself
