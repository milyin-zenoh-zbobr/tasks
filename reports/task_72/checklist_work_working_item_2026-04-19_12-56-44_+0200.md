In settings.gradle.kts, the includeBuild("zenoh-java") substitution is unconditional (active whenever the submodule directory exists). The publish workflows (publish-jvm.yml, publish-android.yml) still check out submodules recursively, which activates this substitution during publication, coupling the release to a local submodule build instead of resolving against the published Maven artifact.

Fix: Add a gradle property (e.g. `zenoh.useLocalJniRuntime=true` in gradle.properties, checked via `findProperty`) to gate the includeBuild substitution. Update settings.gradle.kts to only include the submodule build when the property is set. Remove the submodule checkout steps from both publish workflows so they resolve against the published Maven artifact. Local developers can set the property in their local gradle.properties or pass -P flag to use the submodule.
